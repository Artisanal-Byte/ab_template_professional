# AB Template Specs

This directory contains spec packets for **remaining or future** template work only.

## Existing implemented capabilities

All already-built template capabilities are documented in `.ai/template/*`:

- [profile.md](../template/profile.md) — template identity, stack, structure, conventions
- [design-system.md](../template/design-system.md) — visual language, tokens, form controls, layouts, patterns
- [component-inventory.md](../template/component-inventory.md) — full reusable component inventory
- [usage-rules.md](../template/usage-rules.md) — agent-facing rules for template usage
- [change-log.md](../template/change-log.md) — template-level changelog

## What goes here

Only create spec packets in this directory for work that is **not yet done**:

- Missing reusable components that should be in the template
- Missing system-level features (e.g., notification system, test infrastructure)
- Future template enhancements

## Do not backfill

Do not create spec packets for already-completed template features. If a component or system exists in the repo and is documented in `.ai/template/*`, it is considered complete.
